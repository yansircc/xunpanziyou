这是WordPress菜鸟全面指南系列的第一篇文章。

简单地说，WordPress 是一种建站程序，熟悉了它，就能随心所欲做出漂亮且功能强大的网站。

其实市面上除开 WordPress，还有很多建站程序，比如 Wix、Strikingly、Weebly、Drupal、Joomla 等，但为什么说，如果这辈子只能建一个网站，我建议你一定要选 WordPress？

## 全球流行

首先一点，WordPress 的流行程度高，我们学什么东西，就应该捡流行程度高的东西学，就好比，学语言就学英语肯定不会错一样。

学习一门全世界最流行的技术非常占便宜，不仅可以自己建网站，而且哪天失业了，也能上国际的外包网站找活儿做，因为这些外包网站上找人做 WordPress 网站的需求实在太多。

最重要的一点，用 WordPress 做的网站，你后期修改起来非常简单，因为懂 WordPress 的人非常多，你能雇佣全世界海量的开发者帮你解决问题。

这里说个小案例。

我微信上一个朋友，他网站是国内某个建站公司用它们自己的系统做的。

有一天，我朋友想给网站加个统计功能，他希望所有的询盘发送后，都能把数据传输到谷歌的分析工具中，所以他就要求建站公司帮他加上这个功能。

结果建站公司却以“实现不了”为理由拒绝了我朋友。

然后我朋友就找到了我，说让我帮帮忙。

但面对这种情况我能有什么办法？我无计可施啊，因为我朋友整个网站都是基于建站公司的一套模板自动生出来的，想要加统计代码，那建站公司整个模板上都要加上这个功能才行。

除了出钱给建站公司，求他们开发个定制化功能，否则没戏。

而且我这个朋友无论向谁求助，结果都是做不到，做不到就是做不到，因为建站公司的这套自用模板实在太小众，没人能一上手就做二次开发。

所以讲，使用不流行的平台做网站非常危险，因为一旦出了问题，除了建站公司就没人能帮你。

而且后期想加什么功能也非常不方便，除了建站公司愿意帮你定制化开发，这个世界也没几个人懂怎么开发了。

但如果我们用 WordPress 来建站就完全不同了，WordPress 因为全世界流行，所以你想添加什么功能，想修改某个地方，灵活性是非常高的，你可以自己弄，也可以雇佣世界级平台上的自由职业者帮你弄，而且成本会低到你不敢想。

所以说，建站前选对平台非常关键，**平台的流行程度有多高，后期解决问题就有多简单。**

而选流行程度低的平台就反过来了，虽然一开始花的钱可能不多，比如我们上淘宝一搜，发现用模板做出来的网站可能只要几百元，但后期出了问题，改起来是巨麻烦的，很多时候你还得装孙子，去求着建站公司帮你加功能才行。

## 生态强大

再其次，WordPress 网站的生态非常强大，就好比很多人不愿意用 Linux，只喜欢用 Windows，别人问他为什么，十有八九他会说，好多 Windows 的软件，在 Linux 上都用不起来。

这就是因为，Windows 的生态非常强大，基于这个操作系统有无数好用的软件，这些好软件就共同构筑起了一套生态。

那为什么 Linux 用的人少？关键原因就是上面的软件太少了，想下载个输入法，必须先熟悉一套下载命令，下载后还需要熟悉一套解压缩和安装的命令，甚至安装输入法的时候，还要先熟悉系统路径的知识……

所以说，Linux 对于普通用户来说，它的生态太弱了。

> ps：Linux 对于运维和开发来说，它的生态还是非常强大的，远比 Windows 强大。

生态和流行程度是个鸡生蛋，蛋生鸡的问题，你也不知道是先有鸡还是先有蛋，就像你不知道是因为 WordPress 太流行，才使得它生态强大，还是因为它生态强大才这么流行一样。

所以对于现在生态就非常强大的 WordPress，不用考虑，直接学就对了，因为你学会后，它生态上所有的东西对会对你的网站有用。

- 你的网站想发邮件？没问题，一个表单插件搞定。
- 你的网站想在线聊天？没问题，一个在线聊天的插件搞定。
- 你的网站不希望中国人访问？没问题，一个屏蔽 IP 的插件搞定。

只要你想得到，WordPress 的生态就能帮你做到，而且最重要的一点，大部分的插件都是免费的。

其实比 WordPress 好用的建站程序我不是没有见过，比如国内就有个非常优秀的建站程序 farbox，**用了两年之后，我又回归到 WordPress，**为什么？因为 farbox 用的人实在太少，生态太弱，我每次想在网站上加功能，只有两个办法。

要么自己研究它的文档，把代码写出来，要么花钱在程序员客栈这样的外包平台上，找个懂它编程语言的程序员，帮我把功能做出来。

简单功能还好，我自己也能花时间做，但稍微复杂一点的功能，就必须请人来做二次开发，而且开发费用动辄好几千……

但回归 WordPress 就不同了，我想要什么功能，在插件商店里一搜，铁定能找到我要的插件，而且大部分都能免费试用，一方面我不用花精力自己去写代码，另一方面我也不用花钱请程序员。

所以说，一个生态强大的平台对于正常用户来说，太重要了，能节省你大量的时间和精力。

很多人因为 WordPress 太流行，或者 WordPress 学习曲线不平滑，做网站不直观，就抵触 WordPress，转而使用 wix、weebly 这样的平台，这种做法其实很亏。

虽然用 wix 或者 weebly 做起网站来非常快，只要拖拽点击两下就能把网站做出来，但后续你要面临无穷无尽的麻烦。

- 想加表单？可以，加钱（而且未必好用）。
- 想加在线聊天？可以，加钱（而且未必好用）。
- 想做 SEO 优化？可以，加钱（而且未必好用）。
- 想屏蔽中国人？抱歉，加钱也做不到。

正可谓**建站一时爽，运营火葬场**……到最后就会出现我的困境，要么找程序员写插件帮你把特殊的功能实现，要么自己花时间研究平台的文档，自己动手实现。

所以说，建站用 WordPress 不会错，超高全球流行度，加上超强的生态，就保证了 WordPress 是一门值得精通的技术。

## 学起来简单

很多人之所以用 wix 或者 Strikingly 建站，最重要的原因就在于这些平台都能“拖拽”。

一段文字，一张图片，可以在它们定制好的画布上随意拖来拖去，拖出来是什么样子，最后网站前台就呈现出什么样子，非常傻瓜式操作。

但有两点，让这些平台的优势荡然无存。

### 不利于 SEO

这些工具之所以能拖拽，说明它在代码层面做了大量妥协。本来一张图片，使用正常布局去写，只要一行代码就能表达清楚，但这些拖拽平台，为了让图片的位置和用户拖拽的位置丝毫不差，使用了大量的代码帮助图片做定位。

本来一行能写清楚的图片代码，他们需要用十几行代码实现，甚至为了位置的精准，还会有很多“空行”。

但 WordPress 就好很多，许多优秀模板里，协助定位的组件本来就不多，比如一个行（row），几个 column（列），就没了，一张图片，最最复杂的情况下，用三行代码协助定位也能搞定。

所以为什么同样是一个网站，WordPress 的排名往往比其他平台要好？就是因为基于 WordPress 平台的优秀模板，让后台的代码变得超级简洁，而且丁是丁，卯是卯，无效代码很少很少。

再加上，WordPress 平台有海量 SEO 插件，不管你是想加什么有助于 SEO 的功能，比如：

- 做 xml 的地图，
- 写 robots 文件
- 加 featured snippets

WordPress 的插件商店都能帮你分分钟搞定。

所以都 0202 年了，还有人用小众平台建网站争排名，不是说完全不可取，只能说，**享受了小众平台的便捷，就要承受小众平台在运营上的不便**，想用小众平台做好 SEO，要花费的努力会是 WordPress 的好几倍。

### WP 中类似的优秀工具

再然后，WordPress 的平台也出现了类似的拖拽工具，比如 [Elementor](//elementor.com/?ref=8220)、[Divi](//www.elegantthemes.com/gallery/divi/) 等。

这些工具的拖拽功能做到了真正的平衡，一方面，能真正让用户拖拽建站，另一方面，也保证了代码的简洁，虽然这些工具的拖拽功能非常克制，只能一行一行地拖，一列一列地拖，但能保证后台代码的简洁和高度统一。

当然啦，想解除这类工具的“封印”也不是不行，只要胆子大，把所有元素设置成相对定位，那确实能实现页面内任意元素拖来拖去。

但相应的代价和其他小众平台一样，你要承受得住页面布局一团糟，后台代码一团糟的结局……从这个角度看过去，你就知道小众平台的布局和代码质量到底有多堪忧了。

所以这类拖拽工具的涌现，也让 WordPress 更加容易学习，让 WordPress 变得和 Strikingly 一样，拖就完事了，实际上最后学会 WordPress 的人，都会发现 WordPress 和搭积木非常像。
